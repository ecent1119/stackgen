# stackgen

Generate local dev Docker Compose stacks in minutes.

---

## The problem

- Writing `docker-compose.yml` by hand is slow and error-prone
- Dockerfiles require boilerplate you copy-paste every time
- Environment variable wiring breaks silently
- Onboarding new devs means "just run these 14 commands"

---

## What it does

- Interactive TUI â€” select databases, caches, runtimes
- Preset profiles for common stacks (web-app, api, ml, fullstack)
- Multi-stage Dockerfiles with security best practices
- Auto-generated `.env` with secrets and `.env.example` for sharing
- Test container scaffolding (`stackgen test`)

Generated Dockerfile (Go):

```dockerfile
# Go Dockerfile - Generated by stackgen
# Multi-stage build for optimal image size

FROM golang:1.22-alpine AS builder
WORKDIR /app
RUN apk add --no-cache git ca-certificates
COPY go.mod go.sum* ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o /app/server .

FROM alpine:3.19
WORKDIR /app
RUN adduser -D -g '' appuser
USER appuser
COPY --from=builder /app/server /app/server
EXPOSE 8080
CMD ["/app/server"]
```

## Preset profiles

| Profile | Stack |
|---------|-------|
| `web-app` | Node.js + Postgres + Redis |
| `api` | Go + Postgres |
| `ml` | Python + Postgres + Redis |
| `fullstack` | Node + Go + Postgres + Redis + Neo4j |
| `java-enterprise` | Spring Boot + Postgres + Redis |
| `dotnet` | C# + SQL Server |
| `rust-api` | Rust + Postgres + Redis |

Usage: `stackgen init --profile web-app`

## Scope

- Local development and testing only
- Read-only analysis of existing files (when using `add`)
- No production deployment
- No cloud provider integration
- No telemetry, no network calls
